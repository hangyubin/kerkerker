# TV端前端技术方案

## 1. 项目分析

### 1.1 现有项目结构

现有项目是一个基于 Next.js 16.1.1 + React 19.2.0 + TypeScript 的影视资源管理和播放系统，主要功能包括：

- **影视资源管理**：列表获取、详情查看、分类管理、搜索功能
- **播放功能**：多播放源选择、本地HLS播放器、iframe播放器、弹幕功能
- **用户体验**：主题切换、搜索历史、观看历史、响应式设计
- **管理功能**：视频源管理、播放器配置、数据库设置、Dailymotion频道管理

### 1.2 核心API接口

| API路径 | 方法 | 功能描述 |
|---------|------|----------|
| `/api/drama/list` | POST | 获取影视列表 |
| `/api/drama/detail` | POST | 获取影视详情 |
| `/api/drama/categories` | POST | 获取分类列表 |
| `/api/drama/parse` | POST | 解析视频链接 |
| `/api/vod-sources` | POST | 获取视频源列表 |
| `/api/player-config` | GET | 获取播放器配置 |
| `/api/health` | GET | 健康检查 |

### 1.3 数据结构

- **Drama**：影视基本信息（id, name, pic, type, year等）
- **DramaDetail**：影视详情（包含episodes数组）
- **Episode**：剧集信息（name, url）
- **VodSource**：视频源信息（key, name, api, playUrl等）
- **Category**：分类信息（id, name, icon等）

### 3.2 核心组件设计

#### 3.2.1 布局组件

- **Layout**：根布局，包含头部和底部导航
- **Header**：顶部导航栏，包含logo、搜索按钮和设置按钮
- **Footer**：底部导航栏，包含首页、分类、搜索、我的等选项
- **FocusManager**：焦点管理器，处理遥控器导航

#### 3.2.2 首页组件

- **HeroBanner**：英雄横幅，展示热门影视
- **CategoryRow**：分类行，展示不同分类的影视
- **DramaCard**：影视卡片，展示影视封面和基本信息

#### 3.2.3 搜索组件

- **SearchInput**：搜索输入框，支持遥控器输入
- **SearchResults**：搜索结果列表
- **SearchHistory**：搜索历史记录

#### 3.2.4 详情组件

- **DramaInfo**：影视信息展示
- **EpisodeList**：剧集列表，支持遥控器选择
- **SourceSelector**：播放源选择器

#### 3.2.5 播放器组件

- **Player**：主播放器组件
- **DanmakuPanel**：弹幕面板
- **PlayerControls**：播放器控制栏

### 3.3 状态管理

使用 React Context API 进行状态管理，主要包含：

- **ThemeContext**：主题管理（深色/浅色）
- **PlayerContext**：播放器状态管理
- **SearchContext**：搜索状态管理
- **HistoryContext**：历史记录管理

### 3.4 路由设计

| 路由 | 组件 | 功能 |
|------|------|------|
| `/` | HomePage | 首页 |
| `/browse/[type]` | BrowsePage | 分类浏览 |
| `/search` | SearchPage | 搜索页面 |
| `/detail/[id]` | DetailPage | 影视详情 |
| `/play/[id]` | PlayPage | 播放器页面 |

## 4. TV端交互设计

### 4.1 焦点管理

- **遥控器导航**：支持方向键（上、下、左、右）导航
- **焦点高亮**：当前焦点元素显示高亮效果
- **自动滚动**：当焦点靠近边缘时自动滚动
- **焦点记忆**：返回页面时保持上次焦点位置

### 4.2 操作流程

1. **首页**：使用方向键浏览分类和影视卡片
2. **分类页**：使用方向键浏览分类列表和影视卡片
3. **搜索页**：使用遥控器输入搜索关键词，方向键选择搜索结果
4. **详情页**：使用方向键查看影视信息和选择剧集
5. **播放页**：使用遥控器控制播放器（播放/暂停、音量、进度条）

### 4.3 响应式设计

- **屏幕适配**：支持不同尺寸的TV屏幕
- **字体大小**：根据屏幕尺寸自动调整字体大小
- **布局调整**：根据屏幕尺寸调整布局结构

## 5. 性能优化

### 5.1 加载优化

- **懒加载**：图片和组件懒加载
- **预加载**：预加载即将播放的视频
- **缓存**：缓存API响应和图片

### 5.2 渲染优化

- **虚拟列表**：长列表使用虚拟滚动
- **防抖**：搜索输入防抖
- **节流**：滚动事件节流

### 5.3 网络优化

- **压缩**：压缩静态资源
- **CDN**：使用CDN加速静态资源
- **批量请求**：合并多个API请求

## 7. 技术挑战和解决方案

### 7.1 焦点管理

**挑战**：TV端遥控器导航需要精确的焦点管理
**解决方案**：实现自定义焦点管理器，支持方向键导航和焦点记忆

### 7.2 播放器适配

**挑战**：不同TV设备的播放器兼容性
**解决方案**：使用ArtPlayer和HLS.js，确保在不同设备上的播放体验

### 7.3 性能优化

**挑战**：TV端设备性能有限
**解决方案**：实现懒加载、预加载和缓存策略，优化渲染性能

### 7.4 响应式设计

**挑战**：不同尺寸TV屏幕的适配
**解决方案**：使用Tailwind CSS的响应式类，实现自适应布局

本技术方案基于现有项目的核心功能，为TV端设计了一套完整的前端解决方案。通过选择合适的技术栈和架构设计，确保TV端应用具有良好的用户体验和性能表现。同时，考虑到TV端的特殊交互方式，实现了专门的焦点管理和遥控器操作支持，为用户提供流畅的TV端观影体验。
